# روز ۰۲: سه ابزار ضروری Git برای پلتفرم ویندوز

برای شروع استفاده از کنترل نسخه Git، ابتدا باید ابزارهای مناسب Git را نصب کنید. این سری مقالات عمدتاً بر پلتفرم ویندوز تمرکز دارد. این مقاله سه ابزار کنترل نسخه Git را که اغلب از آن‌ها استفاده می‌کنیم معرفی می‌کند، شباهت‌ها و تفاوت‌های این ابزارها را بیان می‌کند و توضیح می‌دهد که چه زمانی از کدام ابزار استفاده کنید.

## دومین ابزار: [GitHub for Windows](https://windows.github.com/)

این ابزاری است که به طور رسمی توسط [GitHub](https://github.com) برای افرادی که درک عمیقی از Git ندارند توسعه یافته است. این ابزار از یک رابط گرافیکی GUI ساده و دوستانه استفاده می‌کند و تلاش می‌کند مفاهیم سخت و پارامترهای دستوری Git را پنهان کند. اگرچه ابزار GitHub for Windows بسیار دوستانه‌تر از Git for Windows است، اما برای افرادی که هیچ مفهومی از Git ندارند، شروع کار با ابزار GitHub for Windows ممکن است آسان نباشد.

> ابزار [GitHub for Windows](https://windows.github.com/) دارای یک ابزار داخلی به نام [Git Shell](https://github.com/dahlbyk/posh-git/) است که از [Git for Windows](https://msysgit.github.io/) استفاده می‌کند، بنابراین اگر Git for Windows را نصب نکرده باشید، نمی‌توانید از ابزار Git Shell در GitHub for Windows استفاده کنید.

این ابزار می‌تواند به شما در انجام کارهای زیادی در زمینه کنترل نسخه Git کمک کند. حداقل عملکردهای اصلی کنترل نسخه ارائه شده است، عمدتاً با قابلیت‌های زیر:

* Clone کردن مخازن (Repositories)
	* Clone کردن مخازن از GitHub (دستور git clone)
* مرور تاریخچه (History)
	* مرور تاریخچه نسخه‌های مخازن محلی (local repositories)
* Commit کردن تغییرات
	* Commit کردن تغییرات در مخازن محلی
* شاخه‌سازی کد (Branch Code)
	* ایجاد و مدیریت شاخه‌های Git، و امکان سوئیچ کردن شاخه‌ها در دایرکتوری کاری (working directory)
* اشتراک‌گذاری در GitHub.com
	* همگام‌سازی تغییرات با GitHub
	* معنای "Sync" در اینجا ادغام git pull و git push با هم است.
	* وقتی GitHub for Windows همگام‌سازی می‌کند، به طور پیش‌فرض از rebase برای ادغام (merge) استفاده می‌کند.

بسیاری از کارهایی که Git for Windows می‌تواند انجام دهد، در GitHub for Windows امکان‌پذیر نیست! اگر واقعاً می‌خواهید آن را در پروژه‌های توسعه بزرگتر به کار ببرید، ممکن است لازم باشد در مورد مناسب بودن آن فکر کنید، اما برای کسانی که توسعه‌دهنده نرم‌افزار نیستند، کارهای ساده کنترل نسخه باید بیش از حد کافی باشد.

## سومین ابزار: [SourceTree](https://www.sourcetreeapp.com/)

[SourceTree](https://www.sourcetreeapp.com/) در حال حاضر ابزاری است که من به شدت توصیه می‌کنم و می‌تواند از پلتفرم‌های ویندوز و مک پشتیبانی کند. این ابزار یک ابزار GUI است که توسط Atlassian (شرکت سازنده JIRA، Confluence، Bamboo) توسعه یافته و به صورت رایگان ارائه می‌شود. با این ابزار، تقریباً تمام کارهای کنترل نسخه Git را می‌توان انجام داد (از جمله جریان کاری git-flow)، که آن را به یک ابزار کنترل نسخه بسیار قدرتمند تبدیل می‌کند.

> [SourceTree](https://www.sourcetreeapp.com/) همچنین نیاز دارد که ابتدا [Git for Windows](https://msysgit.github.io/) نصب شده باشد، زیرا برای کار کردن به ابزارهای خط فرمان Git متکی است.

## خلاصه امروز

امروز ما به طور خلاصه سه ابزار معمول کنترل نسخه Git را در پلتفرم ویندوز معرفی کردیم. اولین مورد [Git for Windows](https://msysgit.github.io/)، دومین مورد [GitHub for Windows](https://windows.github.com/) و سومین مورد [SourceTree](https://www.sourcetreeapp.com/) است. این سه ابزار هر کدام مزایای خود را دارند، اما شخصاً فکر می‌کنم اگر می‌خواهید کنترل نسخه Git را عمیقاً درک کنید، باید از Git for Windows استفاده کنید و یاد بگیرید که دستورات Git را از طریق خط فرمان اجرا کنید. اگر یک ابزار GUI بصری‌تر می‌خواهید، از [SourceTree](https://www.sourcetreeapp.com/) استفاده کنید.

من می‌خواهم تأکید کنم که برای تسلط واقعی بر کنترل نسخه Git، باید درک کنید که Git در لایه زیرین چه کاری انجام می‌دهد. بنابراین در مقالات بعدی، من اساساً عملیات کنترل نسخه Git را با استفاده از [Git for Windows](https://msysgit.github.io/) معرفی خواهم کرد و از [SourceTree](https://www.sourcetreeapp.com/) به عنوان یک ابزار کمکی برای کمک به درک بهتر عملیات کنترل نسخه Git و تغییرات در مخزن (repository) استفاده خواهم کرد.

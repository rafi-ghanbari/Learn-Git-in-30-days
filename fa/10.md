# روز ۱۰: درک نام‌های مطلق اشیاء Git (Git Object Absolute Names)

در فرآیند کنترل نسخه Git، هر نسخه نشان‌دهنده یک شیء کامیت (commit object) است. از آنجایی که در طول کنترل نسخه اغلب شاخه‌ها ایجاد می‌شوند، نمودار کامیت حاصل می‌تواند بسیار پیچیده باشد. بنابراین، نحوه شناسایی نسخه‌های مختلف یا یافتن سریع نسخه‌های خاص برای به دست آوردن اطلاعات بسیار مهم می‌شود. این مقاله به شما کمک می‌کند تا مفاهیم نامِ مطلق اشیاء و روش‌های استفاده رایج آن‌ها را در Git درک کنید.

## نام‌های مطلق اشیاء (Object Absolute Names)

در Git، هر شیء دارای یک id است که توسط هش SHA1 محاسبه می‌شود و این id همان "نام مطلق" است. اگر شیء یک شیء کامیت باشد، پس این نام مطلق همان شیء کامیت است. بنابراین تا زمانی که نام مطلق شیء کامیت را پیدا کنیم، می‌توانیم در هر زمان به آن نسخه دسترسی داشته باشیم. همانطور که با فلش در شکل زیر نشان داده شده است، این commit object id است که ما از طریق `git log` به دست آورده‌ایم. ما می‌توانیم در هر زمان اطلاعات دقیق آن شیء کامیت را به دست آوریم.

![image](figures/10/01.png)

اگر بخواهیم محتوای شیء کامیت را در شکل بالا ببینیم، می‌توانیم از `git cat-file -p commitid` برای دریافت آن استفاده کنیم، همانطور که در شکل زیر نشان داده شده است:

![image](figures/10/02.png)

## سینتکس کوتاه برای نام‌های مطلق اشیاء

از آنجایی که id شیء Git نتیجه هش SHA1 از محتوا است، بسیار طولانی است (۴۰ کاراکتر). هنگام تعیین "نام مطلق" در Git، می‌توانید به جای آن فقط از چند کاراکتر اول با حداقل ۴ کاراکتر استفاده کنید. این بدان معنی است که نام‌های مطلقی با طول بین ۴ تا ۴۰ کاراکتر همگی می‌توانند برای شناسایی یک شیء استفاده شوند.

به عنوان مثال، در شکل بالا نیز می‌بینیم که محتوای این شیء کامیت شامل یک شیء درختی (tree object) است که آن هم دارای نام مطلق `07c1321be49815d53eb2413f0ad5286010ebb6cc` می‌باشد. بنابراین من می‌توانم دوباره از `git cat-file -p treeid` برای دریافت محتوای آن شیء درختی استفاده کنم، همانطور که در شکل زیر نشان داده شده است:

![image](figures/10/03.png)

گاهی اوقات می‌خواهیم سوابق نسخه را دریافت کنیم و از دستور `git log` استفاده می‌کنیم که نتایج مقایسه تغییرات برای هر فایل را نیز خروجی می‌دهد. نتایج بسیار طولانی خواهد بود. در این زمان، می‌توانید از دستور `git log --pretty=oneline` استفاده کنید تا سوابق تاریخی مختصرتری به دست آورید و همچنین می‌توانید "نام مطلق" کامل شیء کامیت را به دست آورید، همانطور که در شکل زیر نشان داده شده است:

![image](figures/10/04.png)

تکنیک رایج دیگر این است که فقط بخشی از "نام مطلق" را خروجی دهید. می‌توانید این کار را از طریق دستور `git log --pretty=oneline --abbrev-commit` اجرا کنید، همانطور که در شکل زیر نشان داده شده است:

![image](figures/10/05.png)

## خلاصه امروز

موارد فوق نحوه استفاده از نام‌های مطلق است. به دلیل ویژگی‌های اشیاء Git، شما می‌توانید از طریق نام مطلق شیء، به هر شیئی در مخزن Git دسترسی داشته باشید. دستورات git بسیار بیشتری وجود دارند که از نام‌های مطلق استفاده می‌کنند. با آشنایی با این مفاهیم و عبارات، بر اشیاء Git تسلط بهتری پیدا خواهید کرد.

اجازه دهید دستورات گیت و پارامترهای یاد گرفته شده امروز را دوباره سازماندهی کنم:

* git log
* git cat-file -p [object_id]
* git log --pretty=oneline
* git log --pretty=oneline --abbrev-commit

# تسلط بر Git در ۳۰ روز

نویسنده: Will Huang (Will 保哥)

![Git](figures/README/01.png)

## مقدمه

در حوزه توسعه نرم‌افزار، کنترل نسخه (version control) کد منبع موضوع بسیار مهمی است. برخلاف سیستم‌های کنترل نسخه متمرکز مانند Subversion یا TFVC (Team Foundation Version Control)، سیستم Git یک سیستم کنترل نسخه توزیع شده (DVCS; Distributed Version Control System) است که مزایای بسیاری را برای کنترل نسخه به ارمغان می‌آورد و نقاط ضعف کنترل نسخه متمرکز سنتی را حل می‌کند؛ از جمله پشتیبانی از عملیات محلی (local)، پشتیبان‌گیری آسان، شاخه‌سازی (branching) و ادغام (merging) قدرتمند و منعطف و موارد دیگر. با این حال، از آنجایی که کنترل نسخه Git هم در مفاهیم کنترل نسخه و هم در استفاده از ابزار با ابزارهای کنترل نسخه متمرکز سنتی تفاوت زیادی دارد، منحنی یادگیری (learning curve) قابل توجهی ایجاد می‌کند. در طول ۳۰ روز آینده، امیدوارم به شما کمک کنم تا درک کاملی از ماهیت کنترل نسخه Git به دست آورید و تجربه یادگیری خود را همراه با مشکلات و راه‌حل‌هایی که در طول انتقال شرکتمان از Subversion به Git با آن‌ها روبرو شدیم، به اشتراک بگذارم.

> **یادداشت ترجمه:** این نسخه انگلیسی (و اکنون فارسی) مفاهیم اصلی و دقت فنی نسخه اصلی چینی سنتی را حفظ کرده و در عین حال برخی محتواها را برای شیوه‌های مدرن Git و مخاطبان بین‌المللی تطبیق داده است. برخی فصل‌ها با مثال‌های اضافی و بهترین شیوه‌های فعلی بهبود یافته‌اند.

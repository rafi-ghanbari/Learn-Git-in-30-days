# روز ۲۴: استفاده از مخزن راه دور GitHub - شروع کار

گیت‌هاب (GitHub) در حال حاضر پرکاربردترین پلتفرم مدیریت آنلاین گیت در جهان است. این پلتفرم شامل پیاده‌سازی کامل مخزن راه دور گیت، مکانیزم‌ها و گزارش‌های کامل ردیابی مسائل (issue tracking) و هزاران پروژه متن‌باز است که از GitHub برای کنترل نسخه گیت استفاده می‌کنند. حتی در بسیاری از شرکت‌ها، GitHub به عنوان پلتفرم اصلی گیت پذیرفته شده است. این مقاله شما را با ابتدایی‌ترین کاربردهای GitHub آشنا می‌کند.

## گیت‌هاب (GitHub) چیست؟

گیت‌هاب یک پلتفرم مبتنی بر وب است که:
* مخازن گیت را در فضای ابری میزبانی می‌کند.
* ابزارهای همکاری (issues، pull requests، project boards) را فراهم می‌کند.
* از طریق GitHub Actions قابلیت CI/CD را ارائه می‌دهد.
* امکان بازبینی کد و گفتگو را فراهم می‌سازد.
* میزبانی رایگان برای پروژه‌های متن‌باز ارائه می‌دهد.
* به عنوان یک پلتفرم اجتماعی برای توسعه‌دهندگان و پروژه‌های متن‌باز عمل می‌کند.

## ایجاد حساب کاربری در GitHub

۱. به آدرس https://github.com بروید.
۲. روی "Sign up" کلیک کنید.
۳. مراحل ثبت‌نام را دنبال کنید.
۴. آدرس ایمیل خود را تایید کنید.

## ایجاد یک مخزن جدید در GitHub

دو رویکرد اصلی هنگام ایجاد مخزن GitHub وجود دارد:

۱. **مخزن خالی (بدون کامیت اولیه)**: بهترین حالت زمانی است که یک مخزن محلیِ موجود برای Push کردن دارید.
۲. **مخزن مقداردهی شده (با کامیت اولیه)**: گیت‌هاب فایل‌های README، .gitignore و/یا License را برای شما ایجاد می‌کند.

### ایجاد یک مخزن خالی

۱. روی آیکون "+" در گوشه سمت راست بالا کلیک کنید.
۲. "New repository" را انتخاب کنید.
۳. موارد زیر را پر کنید:
   * **Repository name** (نام مخزن)
   * **Description** (توضیحات اختیاری)
   * **Public or Private** (عمومی یا خصوصی)
   * گزینه "Initialize this repository with a README" را **تیک نزنید**.
   * .gitignore و license را روی "None" بگذارید.
۴. روی "Create repository" کلیک کنید.

بعد از ایجاد، GitHub دستورالعمل‌های Push کردن یک مخزن موجود را نشان می‌دهد.

### ایجاد یک مخزن مقداردهی شده

۱. مراحل ۱ تا ۳ فوق را دنبال کنید.
۲. گزینه "Initialize this repository with a README" را **تیک بزنید**.
۳. به صورت اختیاری انتخاب کنید:
   * قالب **.gitignore** برای زبان برنامه نویسی خود.
   * **License** (مجوز) برای پروژه خود.
۴. روی "Create repository" کلیک کنید.

گیت‌هاب یک کامیت اولیه با فایل‌های انتخاب شده ایجاد خواهد کرد.

## اتصال مخزن محلی به GitHub

### روش ۱: Push کردن یک مخزن موجود

```
# اضافه کردن ریموت (remote)
git remote add origin https://github.com/username/repo-name.git

# ارسال به GitHub
git push -u origin main
```

### روش ۲: کلون کردن از GitHub

```
# کلون کردن مخزن
git clone https://github.com/username/repo-name.git

# رفتن به دایرکتوری پروژه
cd repo-name
```

## درک Remoteها

ریموت (remote) ارجاعی به مخزنی است که در جای دیگری میزبانی می‌شود. دستورات رایج:

```
# لیست کردن ریموت‌ها
git remote -v

# اضافه کردن ریموت
git remote add <name> <url>

# حذف ریموت
git remote remove <name>

# تغییر نام ریموت
git remote rename <old> <new>

# تغییر URL ریموت
git remote set-url origin <new-url>
```

## Push کردن به GitHub

ارسال کامیت‌های خود به GitHub:

```
# ارسال به شاخه پیش‌فرض
git push origin main

# ارسال تمام شاخه‌ها
git push --all

# ارسال تگ‌ها
git push --tags
```

پرچم `-u` ردیابی بالادستی (upstream tracking) را تنظیم می‌کند:

```
git push -u origin main
# بعد از این، می‌توانید فقط از دستور زیر استفاده کنید:
git push
```

## Pull کردن از GitHub

دریافت آپدیت‌ها از GitHub:

```
# فقط دریافت تغییرات
git fetch origin

# دریافت و ادغام (Merge)
git pull origin main

# اگر upstream تنظیم شده باشد
git pull
```

## روش‌های احراز هویت (Authentication)

### HTTPS

نیاز به نام کاربری و رمز عبور/توکن (token) دارد:

```
https://github.com/username/repo-name.git
```

### SSH

نیاز به تنظیم کلید SSH دارد:

```
git@github.com:username/repo-name.git
```

تنظیم کلیدهای SSH:

```
# تولید کلید SSH
ssh-keygen -t ed25519 -C "your.email@example.com"

# اضافه کردن به ssh-agent
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# کپی کردن کلید عمومی
cat ~/.ssh/id_ed25519.pub
# این کلید را در تنظیمات GitHub اضافه کنید
```

### Personal Access Token

برای احراز هویت HTTPS:

۱. به GitHub Settings > Developer settings > Personal access tokens بروید.
۲. توکن جدید بسازید.
۳. موقع Push کردن، از توکن به جای رمز عبور استفاده کنید.

* راهنمای مشارکت (Contributing)
* اطلاعات مجوز (License)

## ویژگی‌های GitHub

### Issues

رهگیری باگ‌ها، بهبودها و وظایف:

* عناوین توصیفی ایجاد کنید.
* مراحل بازتولید باگ را ارائه دهید.
* از برچسب‌ها (labels) برای سازماندهی استفاده کنید.
* به کامیت‌ها و Pull Request‌ها ارجاع دهید.

### Wiki

مستندات پروژه شما:

* راهنمای شروع به کار
* مستندات API
* سوالات متداول (FAQs)
* نمودارهای معماری

### Projects

سازماندهی کار با بوردهای Kanban:

* To Do (برای انجام)
* In Progress (در حال انجام)
* Done (انجام شده)

### Releases

نشان گذاری نقاط عطف مهم:

۱. به بخش "Releases" بروید.
۲. روی "Draft a new release" کلیک کنید.
۳. یک تگ انتخاب کنید.
۴. یادداشت‌های انتشار را بنویسید.
۵. در صورت نیاز فایل‌های باینری را ضمیمه کنید.

## بهترین تجربه‌های گردش کار در GitHub

۱. **مکرراً Commit کنید**: کامیت‌های کوچک و منطقی ایجاد کنید.
۲. **پیام‌های خوب بنویسید**: توضیح دهید چرا تغییر دادید، نه فقط چه چیزی را.
۳. **قبل از Push ابتدا Pull کنید**: همیشه اول آخرین تغییرات را بگیرید.
۴. **از شاخه‌ها استفاده کنید**: مستقیماً روی شاخه اصلی (main) کامیت نکنید.
۵. **بازبینی کد**: از Pull Request برای اعمال تغییرات استفاده کنید.

## کارهای رایج

### فورک کردن یک مخزن (Forking)

۱. روی دکمه "Fork" در صفحه مخزن کلیک کنید.
۲. فورکِ خود را کلون کنید:
   ```
   git clone https://github.com/your-username/repo-name.git
   ```
۳. ریموت بالادستی (upstream) را اضافه کنید:
   ```
   git remote add upstream https://github.com/original-owner/repo-name.git
   ```

### همگام‌سازی فورک (Syncing)

```
git fetch upstream
git checkout main
git merge upstream/main
git push origin main
```

## خلاصه امروز

گیت‌هاب پلتفرم قدرتمندی برای میزبانی مخازن گیت و همکاری با دیگران فراهم می‌کند. درک نحوه اتصال مخزن محلی به GitHub، ارسال و دریافت تغییرات، و استفاده از ویژگی‌های GitHub برای توسعه نرم‌افزار مدرن ضروری است.

اجازه دهید دستورات گیت و پارامترهای یاد گرفته شده امروز را دوباره سازماندهی کنم:

* git remote add
* git remote -v
* git remote remove
* git remote rename
* git push -u origin main
* git push --all
* git push --tags
* git pull origin main
* git fetch origin
* git clone
